# Internationalization

OpenStoreShot is built for app teams that publish store assets across markets. The UI ships with a first i18n layer for:

- `ja-JP`
- `en`
- `zh-CN`
- `zh-TW`
- `ko`
- `es`
- `fr`
- `de`
- `pt-BR`

## Translation Surfaces

| Surface | Location | Status |
| --- | --- | --- |
| Studio UI chrome | `apps/web/lib/i18n.ts` | Active |
| README | `README.md`, `README.ja-JP.md` | Active |
| Product docs | `docs/*.md` | English source for now |
| Store copy inside projects | `storeshot.project.json` `localeText` | Active |

## Adding a Language

1. Add the BCP-47 code to `locales` in `apps/web/lib/i18n.ts`.
2. Add the native label to `localeLabels`.
3. Add translations to `partialMessages`.
4. Add assertions to `apps/web/lib/i18n.test.ts`.
5. If translating docs, create `README.<locale>.md` and link it from the README language switcher.

Keep commands, file paths, package names, and API names untranslated. Translate labels, explanations, and user-facing task language.

## Fallback Rule

Japanese is the product source language today because the current product review loop is Japanese-first. English should become the long-term source language once the public API stabilizes. Missing secondary strings currently fall back to Japanese so contributors can ship partial locale improvements safely.
